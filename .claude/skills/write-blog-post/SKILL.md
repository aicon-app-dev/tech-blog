# テックブログ記事作成スキル

Aiconのテックブログ記事を作成する。

## 記事のトーン

- **実践ベース**: 理論より「実際にやってみた」「こうなった」
- **正直**: うまくいかなかったことも書く。失敗談は読者に刺さる
- **具体的**: コード例、数値、スクリーンショットを入れる
- **カジュアル**: 堅すぎない。スタートアップのエンジニアが書いてる感じ
- **短め**: 2000〜4000文字。ダラダラ書かない

## 記事の構成テンプレート

```markdown
# タイトル（具体的に。「〜してみた」「〜を解決した話」系が強い）

## TL;DR
- 3行で要約

## 背景・課題
- なぜこれをやることになったか

## やったこと
- 技術的に何をしたか（コード例あり）
- なぜその選択をしたか

## 結果
- Before/After の数値比較

## 学び・反省
- うまくいったこと、いかなかったこと

## まとめ
- 1-2行で締め
```

## 実行手順

### Step 1: テーマの確認
ユーザーにテーマを確認。曖昧な場合は以下を提案:
- AI開発体制（ハーネスエンジニアリング、Claude Code、CLAUDE.md設計）
- 査定ロジック（ブランド品査定アルゴリズム、SBA/エコリングデータ活用）
- アーキテクチャ（Clean Architecture、依存方向の制御）
- DevOps（GitHub Actions + AI、フルオートパイプライン）
- スタートアップのAI活用

### Step 2: ネタの深掘り
関連するコードやナレッジを読む:
- `assessment-system/` のコード・CLAUDE.md
- `personal-tools/knowledge/docs/` のナレッジ

### Step 3: 構成案の提示
見出し + 各セクション1行要約をユーザーに提示して合意を取る。

### Step 4: 本文の執筆
合意した構成に沿って `blog.md` に書く。

### Step 5: レビュー
ユーザーのフィードバックを反映。

## 注意事項

- 具体的な売上、査定金額、顧客名は書かない
- APIキー・認証情報は絶対に含めない
- 社内ツールのURLは出さない
- コード例は抜粋・簡略化する
- 採用数値は出さない
